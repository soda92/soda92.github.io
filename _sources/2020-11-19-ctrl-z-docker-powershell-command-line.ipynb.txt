    "# ctrl+z导致powershell中docker命令行意外退出\n",
    "\n",
    "最近经常使用 pwsh（PowerShell Core）连接 docker 访问 PostgreSQL\n",
    "数据库，当我编辑命令的时候，可以使用\n",
    "Ctrl+左右方向键进行左右词间的跳转，或者使用 alt+backspace\n",
    "进行以词为单位的删除。\n",
    "\n",
    "然而，当我多删了一个词的时候，长时间使用文本编辑器的习惯使我按下了\n",
    "ctrl+z，希望进行恢复。这时，意外发生了：docker 命令行退出，返回到了 pwsh\n",
    "prompt。\n",
    "\n",
    "我尝试进行了一些搜索：\n",
    "\n",
    "-   ctrl+z terminate docker terminal\n",
    "-   suspend and recover in powershell\n",
    "-   docker windows “ctrl z”\n",
    "\n",
    "得到了一些相关的内容：\n",
    "\n",
    "-   [alankent.me](https://alankent.me/2017/07/04/windows-powershell-control-z-and-kitemati/)\n",
    "-   [memotut.com](https://memotut.com/solution-of-the-problem-that-ctrl+z-cannot-be-used-in-powershell-in-docker-for-windows-environment-(tentative)-3d0ce/)\n",
    "\n",
    "看起来，ctrl+z 会生成 EOF，这或许是导致问题的原因。相关的 blog post\n",
    "中提到，在 powershell 中可以使用 Legacy\n",
    "mode（使用旧版控制台）来解决。然而，我使用的是 Windows\n",
    "Terminal，没有相关的选项。\n",
    "\n",
    "最终，我想到了一种替代的方法：[自定义快捷键](https://docs.microsoft.com/en-us/windows/terminal/customize-settings/actions)。\n",
    "\n",
    "我在 windows Terminal 的 settings.json 中放置了如下代码：\n",
    "\n",
    "``` json\n",
    "{\n",
    "  \"command\": \"commandPalette\",\n",
    "  \"keys\": \"ctrl+z\"\n",
    "}\n",
    "```\n",
    "\n",
    "当然，也可以使用其他的command；这并不重要。这时，ctrl+z便会执行自定义的动作，而不是产生EOF。这或许会带来一些其他的影响；然而，这确实解决了我的问题。"
